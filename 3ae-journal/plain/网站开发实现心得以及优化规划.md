# 网站开发实现心得以及优化规划

## 学习编程方法

```utf-8
想要熟练掌握一门技术要经历一下五个阶段：
1.冬练三九，夏练三伏。掌握基础知识、技能，并且要有渴求精神  
   eg：学习搭积木  
            1> 你想拿什么样的积木搭出什么的作品（渴求精神）。积木的属性、样式、种类、甚至积木发展史在这一阶段都不是重点。  
            2> 搞明白我为什么做而不是大家都这样做。想想自己打算做多久，自己性格适不适合，自己的兴趣爱好在不在这里。而不是工资高  
            3> 立志，定目标。没人天生适不适合，如果下定决心做，那就认真做。just do it! 慢慢地就会爱上它，性格也会改变
            4> 积累。世上没有天才，只有一万小时定律。付出一生的努力。
2.照猫画虎
   eg: 照着别人的样子垒积木
            1> demo
3.二一添作五
   eg: 自己着手搭积木
            1> 把学来的技术再分配。融入到自己的项目中，直到无法再添加。
4.返璞归真
   eg: 思考把积木搭好
            1> 量变到质变。当自己的技术再分配无法进行下去的时候，就面临着重构甚至是重写。
            2> 方向。代码方面，安全方面，性能方面，兼容性适配方面等一系列问题就会随之出现。做到极致。
            3> 瓶颈。当代码性能方面做到极致时（没有最优只有更优），代码可读性、兼容性等方面就会降低，如何在他们之间取得平衡。
            4> 奇技淫巧。把单方面做到极致。很多时候我们只能做到相对的极致和相对的平衡。
            5> 山重水复疑无路 柳暗花明又一村。有限的空间和时间，创造无限的可能。
5.信手拈来
   eg: 打蛇打七寸
            1> 总结经验再应用
```
